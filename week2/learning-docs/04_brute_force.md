# 완전 탐색 (Brute Force)

## 개요

**완전 탐색(Brute Force)**은 가능한 모든 경우를 빠짐없이 확인하여 답을 찾는 방법입니다.  
가장 단순한 방식이지만, 경우의 수가 많아지면 비효율적일 수 있습니다.

## 핵심 원리

- 모든 경우의 수를 직접 시도
- 이중(또는 다중) 반복문으로 모든 쌍·조합 탐색
- 중복을 피하기 위해 `i < j` 또는 `j = i+1`부터 탐색

## 두 수의 합 문제

```
목표: nums[i] + nums[j] == target 인 (i, j) 쌍 찾기

이중 반복문:
  for i in range(n-1):
    for j in range(i+1, n):  # 중복 방지
      if nums[i] + nums[j] == target:
        pairs.append((i, j))
```

## 시간 복잡도

- O(n²): 모든 쌍을 확인

## 장단점

**장점**
- 구현이 단순함
- 반드시 답을 찾을 수 있음

**단점**
- n이 크면 시간이 많이 걸림

## 대안

- 해시(dict)를 사용하면 평균 O(n)에 해결 가능 (Two Sum 최적화)
