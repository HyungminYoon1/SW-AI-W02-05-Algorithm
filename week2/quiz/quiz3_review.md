3. 다음 코드의 출력 결과를 예측하고, 왜 그렇게 되는지 각각 설명하시오. 

def modify(a, b, c):
    a += 1
    b.append(4)
    c = [100, 200]

x = 10
y = [1, 2, 3]
z = [7, 8, 9]

modify(x, y, z)

print(x)  # (1)
print(y)  # (2)
print(z)  # (3)

# 답변

이 문제는 Python의 인자 전달 방식과 객체의 mutability를 이해하는지 묻는 문제입니다.
핵심은 다음 두 가지입니다.

Python은 call by object reference (또는 call by sharing) 방식으로 인자를 전달한다.

immutable 객체와 mutable 객체의 동작이 다르다

## 실행 과정
초기 상태
x → 10
y → [1,2,3]
z → [7,8,9]

함수 호출: modify(x, y, z)

함수 내부 변수
a → 10
b → [1,2,3]
c → [7,8,9]

## a += 1
int는 immutable 객체입니다. 따라서 새로운 정수 객체 11 생성

## b.append(4)
list는 mutable 객체입니다. 따라서 append는 객체 자체를 수정합니다.
[1,2,3] → [1,2,3,4]

b 와 y 는 같은 리스트 객체를 참조. 따라서 y → [1,2,3,4]

## c = [100, 200]
이 코드는 리스트 수정이 아닙니다.
새 리스트 생성 후 c → [100,200]로 참조만 변경됩니다.
기존 객체 [7,8,9] 는 그대로 남아 있습니다.
따라서 z → [7,8,9] 는 변하지 않습니다.

## 출력 결과
10
[1, 2, 3, 4]
[7, 8, 9]
