# 문제
6. 브라우저에서 여러 개의 탭(또는 창)을 동시에 열 때, 멀티스레드와 멀티프로세스 중 어떤 방식이 더 유리하다고 생각하는가? 본인의 선택과 그 이유를 구체적으로 서술하시오. (성능, 안정성, 자원 관리 측면 등에서 고려할 것)

---

# 답변
잘 모르겠습니다. 다만 사용자가 브라우저에서 여러 개의 창을 띄워놓았다고 해도, 동시에 화면을 열람하는 것은 수십개가 아닌 최대 2개 정도까지일 것입니다. 또한 웹게임이나 동영상 재생 같이 자원 소모가 큰 작업에 대해서 각각의 성능을 유지하면서도 하나의 작업의 오류가 다른 작업에 영향을 미치지 않게 하기 위해서는 별도의 프로세스로 분리해야 할 것 같습니다.

---

# 정답
브라우저에서 여러 탭을 동시에 열 때는 멀티프로세스 방식이 더 유리하다. 각 탭이나 기능을 별도 프로세스로 분리하면, 한 탭에서 오류가 발생해도 다른 탭이나 브라우저 전체에 영향을 덜 주어 안정성이 높아진다. 또한 운영체제가 프로세스별로 자원을 관리할 수 있어 보안과 격리 측면에서도 유리하다. 멀티스레드는 메모리 공유로 인해 자원 효율은 좋을 수 있지만, 한 스레드의 문제가 전체 프로그램에 영향을 줄 가능성이 더 크다. 따라서 브라우저처럼 독립성과 안정성이 중요한 환경에서는 멀티프로세스가 더 적합하다.
