# 문제
4. 현대 컴퓨터 시스템에서 CPU와 메인 메모리 사이에 L1, L2, L3 캐시를 두는 이유를 프로세서와 메모리 간의 성능 격차 관점에서 설명하시오.

---

# 답변
CPU(프로세서)는 레지스터에 올라와 있는 대상만 작업할 수 있습니다. 주메모리에 올라와있는 데이터를 CPU 레지스터로 옮기기 위해서 물리적 통로인 버스를 지나야 하는데 그 시간이 오래 걸립니다. 따라서 CPU 내부 또는 근접한 곳에 작은 메모리를 담을 수 있는 캐시(L1, L2, L3)를 두어서 주메모리에 있는 데이터 일부를 미리 캐시에 올려두고, CPU가 다음 연산을 하기 위해 필요한 데이터를 주메모리가 아닌 근방의 캐시에서 곧바로 가져올 수 있게 합니다. 캐시로 사용되는 메모리는 SRAM을 사용하면 빠르고 성능이 좋지만 DRAM보다 공간을 많이 차지하고 비쌉니다. 따라서 CPU연산을 크게 지연시키지 않을 만큼의 용량으로 만듭니다.

비유를 하자면 CPU가 요리사라고 하면, L1 캐시는 눈 앞의 도마, L2 캐시는 근처의 식자재 선반, L3는 주방의 냉장고, 주메모리는 다른 건물에 위치한 식료품 창고, 하드 디스크는 거리가 수 km 떨어진 식자재 마트 정도가 될 것 같습니다.
도마에 있는 재료를 바로 작업할 수 있다고 해서, 도마를 냉장고만큼 크게 만들지도 않고, 당장 작업할 필요가 없는 모든 재료를 도마 위에 올려놓지도 않는 것과 유사한 맥락입니다.

---

# 정답
CPU와 메인 메모리 사이에는 큰 성능 격차가 있기 때문에, CPU가 필요한 데이터를 매번 메인 메모리에서 직접 가져오면 속도가 크게 느려집니다. 그래서 CPU 가까이에 더 빠른 소용량 메모리인 L1, L2, L3 캐시를 두어 자주 쓰는 데이터나 곧 사용할 데이터를 미리 저장해 둡니다. 이렇게 하면 메모리 접근 시간을 줄여 CPU가 덜 기다리게 되어 전체 성능이 향상됩니다.
