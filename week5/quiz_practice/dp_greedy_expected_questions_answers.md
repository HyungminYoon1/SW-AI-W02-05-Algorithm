# DP / Greedy 예상 기출문제 모범답안

이 문서는 이전에 제공한 **DP / Greedy 예상 기출문제**에 대한 모범답안을 정리한 자료입니다.  
가능한 한 **핵심 개념, 정답, 간단한 이유**가 함께 보이도록 구성했습니다.

---

## 1. 개념 구분 문제

1. **동전을 사용해 합 `K`를 만드는 최소 동전 개수 구하기**  
   - **정답:** 일반적으로 DP  
   - **이유:** 동전 체계에 따라 Greedy가 실패할 수 있다. 모든 금액에 대한 최적해를 누적하는 DP가 안전하다.

2. **회의실 사용 시간이 주어질 때 최대한 많은 회의 배정하기**  
   - **정답:** Greedy  
   - **이유:** 종료 시간이 가장 빠른 회의를 선택하는 전략이 최적이다.

3. **계단을 오를 때 한 번에 1칸 또는 2칸씩 올라가며, 각 칸의 점수 합의 최댓값 구하기**  
   - **정답:** DP  
   - **이유:** 각 위치까지의 최적해가 이전 상태들에 의해 결정된다.

4. **여러 작업의 마감 시간이 주어질 때 지각을 최소화하도록 작업 순서 정하기**  
   - **정답:** 보통 Greedy 또는 정렬 기반 전략  
   - **이유:** 마감 시간이나 벌점 기준으로 정렬하는 탐욕 전략이 적용되는 대표 유형이 많다. 다만 문제 조건에 따라 달라질 수 있다.

5. **길이가 `N`인 수열에서 증가하는 부분수열의 최대 길이 구하기**  
   - **정답:** DP  
   - **이유:** 각 위치에서 끝나는 최장 증가 부분수열 길이를 누적 계산할 수 있다.

---

## 2. Greedy 정당성 문제

> “항상 현재 시점에서 가장 좋아 보이는 선택을 하면 전체 최적해를 얻을 수 있다.”

1. **화폐 단위 `{500, 100, 50, 10}`**  
   - **정답:** 참  
   - **이유:** 큰 단위가 작은 단위의 배수 관계를 이루므로 큰 동전부터 선택해도 손해가 없다.

2. **화폐 단위 `{1, 3, 4}`**  
   - **정답:** 거짓  
   - **반례:** 금액 `6`  
   - Greedy: `4 + 1 + 1` → 3개  
   - 최적해: `3 + 3` → 2개

3. **회의실 배정에서 “끝나는 시간이 가장 빠른 회의” 우선**  
   - **정답:** 참  
   - **이유:** 가장 빨리 끝나는 회의를 선택하면 이후에 선택 가능한 회의 수를 최대화할 수 있다. 교환 논법으로 정당화 가능하다.

---

## 3. DP 점화식 세우기

**문제:** 연속하지 않은 원소들만 선택 가능할 때 합의 최댓값

1. **상태 정의**  
   - `dp[i]`: `1`번부터 `i`번 원소까지 고려했을 때 얻을 수 있는 최대 합

2. **점화식**  
   - `dp[i] = max(dp[i-1], dp[i-2] + A[i])`

3. **초기값**  
   - `dp[1] = A[1]`
   - `dp[2] = max(A[1], A[2])`

4. **시간복잡도**  
   - `O(N)`

---

## 4. 0/1 Knapsack 유형

1. **Greedy로 잘 풀리지 않는 이유**  
   - 가치/무게 비율이 높다고 항상 최적은 아니다.  
   - 물건을 쪼갤 수 없으므로 지역 최선이 전체 최선으로 이어지지 않는다.

2. **DP 상태 정의**  
   - `dp[i][j]`: 앞의 `i`개 물건만 고려하고, 최대 무게가 `j`일 때 얻을 수 있는 최대 가치

3. **점화식**  
   - `j < w[i]`이면 `dp[i][j] = dp[i-1][j]`
   - `j >= w[i]`이면 `dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])`

4. **1차원 DP에서 뒤에서 순회해야 하는 이유**  
   - 같은 물건을 한 번만 써야 하는데, 앞에서부터 갱신하면 같은 물건이 한 라운드에서 여러 번 반영될 수 있다.

---

## 5. 최소 동전 개수

화폐 단위 `1, 4, 6`, 금액 `8`

1. **Greedy 결과**  
   - `6 + 1 + 1` → 3개

2. **최적해**  
   - `4 + 4` → 2개

3. **Greedy가 항상 맞지 않는 이유**  
   - 가장 큰 동전을 먼저 고르는 선택이 이후 조합을 망칠 수 있다.

4. **DP 점화식**  
   - `dp[x] = min(dp[x-1] + 1, dp[x-4] + 1, dp[x-6] + 1)`  
   - 단, 가능한 경우만 고려

---

## 6. 활동 선택 문제

1. **정렬 기준**  
   - 종료 시간이 빠른 순

2. **시작 시간이 빠른 순 정렬 반례**  
   - 회의 `(1, 10), (2, 3), (4, 5), (6, 7)`  
   - 시작 시간 기준이면 `(1,10)`을 먼저 선택해 나머지를 거의 못 고른다.

3. **종료 시간이 같은 경우 추가 기준**  
   - 시작 시간이 빠른 순 또는 늦은 순 어느 쪽이든 가능하나, 보통 시작 시간이 빠른 순을 함께 둔다.

4. **시간복잡도**  
   - 정렬 `O(N log N)` + 선택 `O(N)` = 전체 `O(N log N)`

---

## 7. LIS 기초

예시 수열: `10 20 10 30 20 50`

1. **`dp[i]`의 의미**  
   - `i`번째 원소를 마지막으로 하는 LIS의 길이

2. **점화식**  
   - `dp[i] = 1 + max(dp[j])` for all `j < i` and `A[j] < A[i]`
   - 없으면 `dp[i] = 1`

3. **시간복잡도**  
   - 기본 DP는 `O(N^2)`

4. **Greedy만으로 어려운 이유**  
   - 지금 큰 수를 고르는 선택이 미래의 더 긴 증가 수열을 막을 수 있다.

5. **예시 정답**  
   - LIS 길이 = `4`  
   - 예: `10, 20, 30, 50`

---

## 8. 계단 오르기 변형

1. **단순 Greedy가 어려운 이유**  
   - 현재 점수가 큰 계단을 고르는 것이 전체 최댓값을 보장하지 않는다. 연속 3칸 금지 조건 때문에 미래 선택과 얽힌다.

2. **상태 정의 예시**  
   - `dp[i]`: `i`번째 계단에 도달했을 때 얻을 수 있는 최대 점수

3. **점화식**  
   - `dp[i] = max(dp[i-2] + score[i], dp[i-3] + score[i-1] + score[i])`

4. **초기값 주의점**  
   - `i=1,2,3`은 직접 계산해 두어야 한다.
   - 마지막 계단은 반드시 밟아야 한다는 조건을 반영해야 한다.

---

## 9. 분할 불가능한 배낭 vs 분할 가능한 배낭

1. **분할 불가능한 배낭(0/1 Knapsack)**  
   - **적절한 방식:** DP  
   - **이유:** 물건을 쪼갤 수 없어 비율만으로 최적 보장 불가

2. **분할 가능한 배낭(Fractional Knapsack)**  
   - **적절한 방식:** Greedy  
   - **이유:** 가치/무게 비율이 높은 것부터 담으면 최적
   - **정렬 기준:** `v[i] / w[i]` 내림차순

---

## 10. 문자열 DP

팰린드롬 분할 최소 개수 문제

1. **최적 부분 구조가 성립하는 이유**  
   - 어떤 접두사에 대한 최적 분할은 그보다 짧은 접두사의 최적 분할 결과를 이용해 구성 가능하다.

2. **DP 상태 정의 예시**  
   - `dp[i]`: 앞에서부터 `i`번째 문자까지 최소 분할 수

3. **유리한 사전 계산**  
   - `pal[l][r]`: 구간 `l..r`가 팰린드롬인지 여부

4. **시간복잡도**  
   - 팰린드롬 판별 `O(N^2)`  
   - DP `O(N^2)`  
   - 전체 `O(N^2)`

---

## 11. Greedy 반례 만들기

1. **“항상 가장 큰 수를 먼저 선택하면 된다.” 반례**  
   - 동전 `{1,4,6}`, 금액 `8`  
   - 가장 큰 수 `6`을 먼저 선택하면 최적이 아니다.

2. **“현재 비용이 가장 적은 선택을 계속하면 전체 비용도 최소다.” 반례**  
   - 최단 경로/작업 선택 문제에서 초반에 싼 선택이 이후 큰 손해를 유발할 수 있다.

3. **“비율이 가장 높은 것부터 고르면 항상 최적이다.” 반례**  
   - 0/1 Knapsack에서 비율 기준으로 고르면 실패 가능

4. **“먼저 시작하는 구간부터 고르면 최대 개수 선택이 가능하다.” 반례**  
   - `(1,10), (2,3), (4,5), (6,7)`

---

## 12. DP 테이블 해석 문제

`dp = [0, 1, 1, 2, 3, 5, 8, 13]`

1. **가능성이 높은 점화식**  
   - 피보나치형 점화식  
   - `dp[i] = dp[i-1] + dp[i-2]`

2. **유추 가능한 문제 유형**  
   - 계단 오르기 경우의 수  
   - 타일 채우기  
   - 피보나치 수열 자체

3. **같은 DP 배열이라도 의미가 다른 이유**  
   - DP 값은 “수치”만 같을 뿐, 상태 정의가 다르면 전혀 다른 문제를 표현할 수 있다.

---

## 13. 구현형 문제: 거스름돈

1. **Greedy가 가능한 경우 / 불가능한 경우**  
   - 가능한 경우: 화폐 단위가 배수 관계 등으로 정당화 가능한 체계
   - 불가능한 경우: 임의의 동전 체계. 반례가 쉽게 존재

2. **일반적으로 안전한 알고리즘**  
   - DP

3. **시간복잡도**  
   - `N`종류 동전, 목표 금액 `K`일 때 `O(NK)`

4. **대표 DP 점화식**  
   - `dp[0] = 0`
   - `dp[x] = min(dp[x], dp[x-coin] + 1)` for each coin

---

## 14. 구현형 문제: 최대 합 증가 부분수열

1. **`dp[i]` 정의**  
   - `i`번째 원소를 마지막으로 하는 증가 부분수열의 최대 합

2. **LIS와 비슷한 점 / 다른 점**  
   - 비슷한 점: 이전 원소들 중 작은 값에서 전이
   - 다른 점: 길이가 아니라 합을 최적화

3. **점화식**  
   - 초기: `dp[i] = A[i]`
   - 전이: `A[j] < A[i]`이면 `dp[i] = max(dp[i], dp[j] + A[i])`

4. **시간복잡도**  
   - `O(N^2)`

---

## 15. 서술형 종합 문제

1. **핵심 차이**  
   - Greedy: 매 순간 최선의 선택
   - DP: 부분 문제의 최적해를 저장하여 전체 최적해 구성

2. **성립 조건**  
   - Greedy: 탐욕 선택 속성, 최적 부분 구조
   - DP: 최적 부분 구조, 중복 부분 문제

3. **대표 문제**  
   - Greedy: 회의실 배정, Fractional Knapsack
   - DP: LIS, 0/1 Knapsack

4. **Greedy에서 정당성 증명이 중요한 이유**  
   - 빠르더라도 잘못된 전략이면 최적해를 놓칠 수 있기 때문

5. **DP에서 상태 정의가 중요한 이유**  
   - 상태가 올바르지 않으면 점화식을 세울 수 없고, 세워도 올바른 답이 나오지 않는다.

# 실전 미니 모의고사 정답

## A. OX 문제

1. DP 문제는 항상 재귀로만 풀어야 한다.  
   - **X**  
   - 반복문 기반 bottom-up도 가능하다.

2. Greedy는 항상 최적해를 보장하지 않는다.  
   - **O**

3. 최적 부분 구조가 있으면 항상 Greedy로 풀 수 있다.  
   - **X**  
   - DP가 필요한 경우도 많다.

4. 0/1 Knapsack은 일반적으로 DP로 푼다.  
   - **O**

5. 회의실 배정 문제는 대표적인 Greedy 문제다.  
   - **O**

6. LIS는 대표적인 DP 문제다.  
   - **O**

---

## B. 단답형

1. **DP의 두 가지 대표 구현 방식**  
   - Top-down(메모이제이션), Bottom-up(테이블)

2. **Greedy 알고리즘의 핵심 아이디어**  
   - 매 단계에서 가장 좋아 보이는 선택을 한다.

3. **메모이제이션은 무엇을 방지하는가?**  
   - 중복 계산

4. **Greedy가 성립하려면 자주 필요한 두 성질**  
   - 탐욕 선택 속성
   - 최적 부분 구조

5. **1차원 DP 최적화 시 갱신 순서가 중요한 이유**  
   - 현재 단계 값이 같은 단계의 다른 계산에 잘못 재사용될 수 있기 때문이다.

---

## C. 서술형

1. **“Greedy는 빠르지만 위험하고, DP는 안전하지만 느릴 수 있다.”**  
   - Greedy는 보통 정렬이나 선형 탐색으로 빠르게 구현되지만, 정당성이 없으면 오답이 된다.  
   - DP는 많은 경우 최적해를 안정적으로 구할 수 있지만 상태 수가 많아지면 시간과 메모리 부담이 커진다.

2. **“상태를 잘못 정의하면 DP는 시작도 못 한다.”**  
   - 예를 들어 LIS에서 `dp[i]`를 단순히 “앞에서 i개 봤을 때의 답”으로만 두면 전이가 모호해진다.  
   - 반면 “i에서 끝나는 LIS 길이”로 정의하면 전이가 명확해진다.

3. **Greedy에서 반례를 찾는 습관이 중요한 이유**  
   - 그 전략이 진짜 항상 맞는지 빠르게 검증할 수 있기 때문이다.  
   - 작은 반례 하나로 잘못된 알고리즘을 걸러낼 수 있다.

# 난이도 높은 예상 문제 답안

## 16. 최소 점프 횟수

1. **DP로도 풀리는가?**  
   - 예. `dp[i] = i까지 도달하는 최소 점프 수`로 정의 가능

2. **Greedy로 더 효율적으로 풀 수 있는가?**  
   - 예. 현재 점프 횟수로 도달 가능한 범위를 관리하면 `O(N)` 가능

3. **핵심 아이디어**  
   - 현재 범위 안에서 다음에 가장 멀리 갈 수 있는 위치를 추적하다가, 범위를 벗어나는 순간 점프 횟수를 증가시킨다.

---

## 17. 구간 선택 최적화

1. **단순 회의실 배정과의 차이**  
   - 목표가 “개수 최대”가 아니라 “가중치 합 최대”이다.

2. **Greedy로 어려운 이유**  
   - 짧거나 빨리 끝나는 구간이 항상 높은 가중치를 보장하지 않는다.

3. **필요한 DP 접근**  
   - 구간을 끝나는 시간 기준 정렬 후  
   - `dp[i] = i번째 구간까지 고려했을 때 최대 가중치 합`

4. **이전 선택 가능한 마지막 구간을 빨리 찾는 방법**  
   - 이분 탐색

---

## 18. 문자열 편집 거리

1. **왜 DP 문제인가?**  
   - 문자열의 접두사들에 대한 최소 편집 횟수로 전체 문제를 쪼갤 수 있다.

2. **상태 정의**  
   - `dp[i][j]`: A의 앞 `i`글자를 B의 앞 `j`글자로 바꾸는 최소 연산 수

3. **세 가지 전이**  
   - 삽입: `dp[i][j-1] + 1`
   - 삭제: `dp[i-1][j] + 1`
   - 치환 또는 일치:
     - 같으면 `dp[i-1][j-1]`
     - 다르면 `dp[i-1][j-1] + 1`

4. **시간복잡도 / 공간복잡도**  
   - 시간복잡도 `O(NM)`
   - 공간복잡도 `O(NM)`  
   - 최적화 시 `O(min(N,M))`까지 가능
