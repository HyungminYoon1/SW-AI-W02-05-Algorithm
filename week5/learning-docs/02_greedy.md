# 02. 그리디 알고리즘

## 그리디 알고리즘이란?

그리디 알고리즘(Greedy Algorithm)은 문제를 해결하는 과정에서 매 순간 가장 좋아 보이는 선택을 하는 방법입니다.

쉽게 말하면:

- 지금 당장 최선이라고 생각되는 선택을 하고
- 그 선택을 되돌리지 않은 채
- 끝까지 진행해서 답을 만드는 방식입니다.

## 왜 그리디가 매력적인가?

그리디는 생각보다 구현이 단순한 경우가 많습니다.

- 상태를 많이 저장하지 않아도 되는 경우가 많음
- 반복문과 정렬만으로 풀리는 경우가 많음
- 시간 복잡도가 비교적 좋은 경우가 많음

그래서 적용만 가능하다면 매우 효율적인 풀이가 됩니다.

## 하지만 항상 맞는 것은 아니다

그리디에서 가장 중요한 점은:

"지금의 최선 선택이 전체 최적해로 이어지는가?" 입니다.

이 조건이 보장되지 않으면 그리디는 틀릴 수 있습니다.

예를 들어 어떤 문제는 지금 가장 커 보이는 값을 고르면 나중에 더 큰 손해가 생길 수 있습니다. 이런 문제는 DP나 다른 방법이 필요합니다.

## 그리디가 잘 맞는 문제의 특징

### 1. 현재의 최선 선택이 이후에도 유효하다

지금 한 선택이 나중에 발목을 잡지 않아야 합니다.

예:

- 회의실 배정
- 거스름돈 문제(특정 화폐 체계)
- 활동 선택 문제

### 2. 선택 기준이 분명하다

예를 들어:

- 끝나는 시간이 가장 빠른 회의를 먼저 선택
- 가치 대비 무게가 큰 것을 먼저 선택
- 지금 가장 멀리 다시 사용될 전자기기를 빼기

처럼 기준이 명확해야 합니다.

## 대표 예시 1: 회의실 배정

회의가 여러 개 있을 때 최대한 많은 회의를 선택하려면, 보통

- 끝나는 시간이 빠른 회의부터 고릅니다.

왜냐하면 빨리 끝나는 회의를 선택해야 이후에 더 많은 회의를 넣을 여지가 생기기 때문입니다.

이 문제는 대표적인 그리디 예시입니다.

## 대표 예시 2: 거스름돈

동전 단위가 적절하게 주어졌을 때는 가장 큰 동전부터 최대한 사용하는 방식이 맞을 수 있습니다.

예:

- 500원, 100원, 50원, 10원

이런 체계에서는 큰 동전부터 쓰는 것이 자연스럽고 정답이 됩니다.

하지만 모든 동전 체계에서 항상 맞는 것은 아닙니다.

예를 들어 동전이 `1, 3, 4`이고 목표가 `6`이면:

- 그리디: `4 + 1 + 1` -> 3개
- 최적해: `3 + 3` -> 2개

즉 이런 경우는 DP가 필요합니다.

## 대표 예시 3: 멀티탭 스케줄링

앞으로 사용될 기기 순서를 알고 있을 때, 지금 뽑아야 하는 기기를 정해야 합니다.

이때는:

- 다시 가장 늦게 사용되거나
- 앞으로 아예 사용되지 않는 기기

를 뽑는 것이 유리합니다.

이처럼 현재 선택 기준이 미래 전체에 대해 논리적으로 설명될 수 있으면 그리디가 성립합니다.

## 그리디 문제를 풀 때 자주 하는 과정

1. 규칙을 하나 정한다.
2. 정렬이 필요하면 정렬한다.
3. 앞에서부터 보면서 규칙대로 선택한다.
4. 선택이 가능한지 조건을 갱신한다.

많은 그리디 문제는 결국 "무엇을 기준으로 정렬할 것인가?"가 핵심입니다.

## DP와 그리디의 차이

### 그리디

- 매 순간 하나의 선택을 바로 확정
- 보통 되돌아가지 않음
- 빠르고 단순한 경우가 많음

### DP

- 여러 상태를 모두 고려
- 이전 계산 결과를 저장
- 더 일반적이지만 구현이 복잡할 수 있음

즉 그리디는 "하나를 믿고 간다"에 가깝고, DP는 "가능한 상태를 체계적으로 관리한다"에 가깝습니다.

## 그리디가 맞는지 확인하는 방법

그리디는 단순히 직감으로 정하면 위험합니다. 보통 아래를 생각해야 합니다.

- 왜 지금 이 선택이 항상 유리한가?
- 이 선택을 해도 이후 최적해를 잃지 않는가?
- 반례를 만들어 보면 깨지지 않는가?

그리디 문제를 잘 풀려면 정답 코드보다 "왜 이 기준이 맞는가"를 설명할 수 있어야 합니다.

## 자주 헷갈리는 포인트

- 빨라 보인다고 모두 그리디는 아닙니다.
- 현재 최선 선택이 전체 최선으로 이어진다는 근거가 필요합니다.
- 동전 문제는 어떤 것은 그리디, 어떤 것은 DP입니다.
- 정렬 후 순서대로 고르는 문제가 많지만, 정렬 기준이 잘못되면 바로 오답이 됩니다.

## 다른 사람에게 설명할 때 좋은 표현

- "그리디는 매 순간 가장 좋아 보이는 선택을 하는 방식입니다."
- "중요한 것은 그 선택이 전체 최적해를 보장하는지입니다."
- "그리디는 보통 선택 기준과 정렬 기준을 찾는 것이 핵심입니다."
- "반례가 생기면 그리디는 틀리고, DP 같은 다른 접근이 필요할 수 있습니다."

## 정리

그리디 알고리즘은 문제를 매우 빠르고 간단하게 풀 수 있는 강력한 방법이지만, 항상 맞는 것은 아닙니다. 그래서 문제를 볼 때는 "지금의 최선이 정말 전체 최선으로 이어지는가?"를 반드시 따져 봐야 합니다. 그리디는 구현보다 정당화가 더 중요한 알고리즘입니다.
