# 01. 동적 계획법

## 동적 계획법이란?

동적 계획법(Dynamic Programming, DP)은 큰 문제를 작은 문제로 나누어 해결하되, 한 번 계산한 결과를 저장해 두고 다시 활용하는 방법입니다.

핵심 아이디어는 두 가지입니다.

- 같은 하위 문제가 여러 번 반복된다.
- 작은 문제의 답으로 큰 문제의 답을 만들 수 있다.

이 두 조건이 맞을 때 DP가 강력하게 작동합니다.

## 왜 DP가 필요한가?

재귀만으로 문제를 풀면 같은 계산을 여러 번 반복하는 경우가 많습니다.

예를 들어 피보나치 수열을 단순 재귀로 구하면:

- `f(5)`를 구할 때 `f(4)`와 `f(3)`을 계산하고
- `f(4)`를 구할 때 다시 `f(3)`과 `f(2)`를 계산합니다.

즉, `f(3)` 같은 값이 여러 번 반복 계산됩니다.

DP는 이런 중복을 없애기 위해 이미 구한 값을 저장합니다.

## DP를 적용할 수 있는 문제의 특징

### 1. 최적 부분 구조

큰 문제의 최적해가 작은 문제들의 최적해로부터 만들어질 수 있어야 합니다.

예를 들어:

- 최소 비용
- 최대 점수
- 경우의 수

같은 문제들은 종종 작은 문제의 답을 조합해서 해결할 수 있습니다.

### 2. 중복되는 부분 문제

같은 상태를 여러 번 계산하게 되는 구조여야 합니다.

이때 상태를 저장해 두면 시간 복잡도를 크게 줄일 수 있습니다.

## DP를 풀 때 가장 먼저 해야 할 일

DP는 보통 아래 순서로 접근합니다.

1. `dp`가 무엇을 의미하는지 정의한다.
2. 점화식을 세운다.
3. 시작값 또는 종료 조건을 정한다.
4. 계산 순서를 정한다.

이 중에서 가장 중요한 것은 `dp[i]` 또는 `dp[i][j]`가 정확히 무엇을 뜻하는지 정하는 것입니다.

## 상태 정의 예시

예를 들어 계단 오르기 문제라면:

- `dp[i]`: `i`번째 계단까지 왔을 때의 최대 점수

동전 문제라면:

- `dp[x]`: 금액 `x`를 만드는 방법 수

외판원 순회처럼 비트마스크를 쓰는 문제라면:

- `dp[now][visited]`: 현재 `now` 도시에 있고 `visited` 상태일 때 남은 도시를 모두 방문하고 출발점으로 돌아가는 최소 비용

상태 정의가 명확해야 점화식도 자연스럽게 나옵니다.

## 점화식이란?

점화식은 현재 상태의 답을 이전 상태의 답으로 표현한 식입니다.

예를 들어 피보나치 수열은:

```python
dp[i] = dp[i - 1] + dp[i - 2]
```

계단 오르기에서는:

```python
dp[i] = max(dp[i - 2], dp[i - 3] + stairs[i - 1]) + stairs[i]
```

처럼 이전 상태와의 관계를 식으로 나타냅니다.

## DP 구현 방식 1: 메모이제이션

메모이제이션(Memoization)은 재귀를 사용하면서, 이미 계산한 값을 배열이나 딕셔너리에 저장해 두는 방식입니다.

보통 탑다운(Top-Down) 방식이라고 부릅니다.

```python
def dfs(x):
    if x == 0:
        return 0
    if dp[x] != -1:
        return dp[x]

    dp[x] = dfs(x - 1) + 1
    return dp[x]
```

장점:

- 재귀 구조라 문제를 식으로 생각하기 쉬움
- 필요한 상태만 계산할 수 있음

주의할 점:

- 재귀 깊이가 깊어질 수 있음
- 종료 조건을 정확히 잡아야 함

## DP 구현 방식 2: 테이블 채우기

테이블 채우기(Tabulation)는 작은 상태부터 차례대로 `dp` 배열을 채워 나가는 방식입니다.

보통 바텀업(Bottom-Up) 방식이라고 부릅니다.

```python
dp[0] = 0
for i in range(1, n + 1):
    dp[i] = dp[i - 1] + 1
```

장점:

- 반복문 기반이라 재귀 제한 걱정이 적음
- 계산 순서가 명확하면 구현이 단순함

## 1차원 DP와 2차원 DP

### 1차원 DP

상태가 한 개의 기준으로만 나뉘는 경우입니다.

예:

- `dp[i]`: `i`까지의 최소 비용
- `dp[i]`: `i`를 만드는 경우의 수

### 2차원 DP

상태를 나타내기 위해 기준이 두 개 필요한 경우입니다.

예:

- `dp[i][j]`: `i`번째 물건까지 고려했을 때 무게 `j`에서의 최대 가치
- `dp[i][j]`: 문자열 A의 앞 `i`글자와 문자열 B의 앞 `j`글자의 관계

## 경우의 수 DP와 최적화 DP

DP 문제는 크게 두 방향으로 자주 나옵니다.

### 경우의 수 세기

예:

- 동전으로 금액 만들기
- 타일 채우기
- 경로 개수 세기

이때는 보통 `+` 연산으로 누적합니다.

### 최솟값 / 최댓값 구하기

예:

- 최소 비용
- 최대 점수
- 최단 거리 비슷한 최적화 문제

이때는 보통 `min`, `max`를 사용합니다.

## DP 문제를 만났을 때 스스로에게 할 질문

- 이 문제에서 반복 계산되는 상태는 무엇인가?
- 그 상태를 배열 인덱스로 표현할 수 있는가?
- 현재 답이 이전 어떤 상태들로부터 오는가?
- 최소값, 최대값, 경우의 수 중 무엇을 구하는가?
- 시작값과 종료 조건은 무엇인가?

이 질문들에 답할 수 있으면 DP 풀이가 훨씬 선명해집니다.

## 자주 헷갈리는 포인트

- 점화식을 먼저 외우기보다 상태 정의를 먼저 해야 합니다.
- `dp` 배열의 값이 무엇을 뜻하는지 흐려지면 점화식도 틀리기 쉽습니다.
- 메모이제이션과 바텀업은 구현 방식이 다를 뿐, 같은 상태와 점화식을 공유할 수 있습니다.
- 경우의 수를 세는 문제에서는 반복문의 순서에 따라 조합을 세는지 순열을 세는지가 달라질 수 있습니다.

## 다른 사람에게 설명할 때 좋은 표현

- "DP는 같은 계산을 여러 번 하지 않도록 결과를 저장하면서 푸는 방법입니다."
- "핵심은 상태 정의와 점화식입니다."
- "작은 문제의 답을 이용해서 큰 문제의 답을 만들어 갑니다."
- "메모이제이션은 재귀 중심, 바텀업은 반복문 중심이라고 보면 됩니다."

## 정리

동적 계획법은 문제를 작은 상태로 나누고, 그 상태의 답을 저장하면서 최적해나 경우의 수를 효율적으로 구하는 방법입니다. DP를 잘 풀기 위해서는 코드를 바로 짜기보다 먼저 상태 정의와 점화식을 정확히 세우는 습관이 중요합니다.
