## Divide-and-Conquer
1. Devide: 큰 덩어리로 자르기
2. Delegate: 각각의 인스턴스에 재귀 적용
3. Combine: 더 작은 인스턴스를 최종 인스턴스의 해결로 구한다.

예제) 배열에서 가장 큰 수 찾기
1. 배열을 절반으로 나눔
2. 나온 배열을 계속 절반으로 나눔
3. 배열의 원소가 0개 또는 1개가 될 때까지 나누어지면 인접한 배열과 비교하여 최댓값을 추한다
4. 배열을 계속 병합해가면서 최댓값을 갱신한다.

시간복잡도에 대한 점화식:
T(n) = 2T(n/2) + O(1)

## recursion tree에 대한 점화식을 푸는 방법 -> 알아서 학습

recursion tree란? 분할정복 재귀식이 있을 때, 재귀 호출이 트리처럼 퍼지는 구조를 그려서 각 호출의 비용을 더하는 방식

점화식 T(n) = a*T(n/b) + f(n) 일 때

1단계)
깊이 i 에서
- 자식 노드 수: a^i
- 각 자식 문제 크기: n/b^i
- 현재 각 노드의 재귀 외 비용: f(n/b^i)
- 각 노드의 비용은 f(n/bi)
2단계) 레벨 비용: a^i * f(n/b^i)
3단계) 높이: logb(n)
4단계) 합: 시그마 i = 0 부터 i = logb(n-1) 까지 a^i * f(n/b^i)

*깊이 i의 전체 비용은 a^i⋅f(n/b^i)

## 동적계획법
- 동적 계획법은 반복 없는 재귀
- 중간의 부분문제에 대한 솔루션을 저장(기억)하는데 보통 배열이나 테이블을 이용한다.
- 중요한 것은 올바른 점화식을 찾는 것

### 재귀 알고리즘 혹은 점화식을 찾아내야 함

### 동적 계획법 패턴
1. 계산해야할 배열의 값들을 명확히 정의한다
2. 계산할 값의 점화식을 찾는다 -> 재귀적 구조를 찾는다
3. 점화식을 이용하여 배열에 값들을 계산해서 채운다
4. 원래 문제의 답을 추출한다.
