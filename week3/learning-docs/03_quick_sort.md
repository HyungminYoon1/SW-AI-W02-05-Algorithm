# 퀵 정렬 (Quick Sort)

## 개요

**퀵 정렬**은 분할 정복 방식을 사용하는 정렬 알고리즘입니다. 피벗(pivot)을 하나 선택하고, 피벗보다 작은 값은 왼쪽, 큰 값은 오른쪽으로 분할한 뒤 재귀적으로 정렬합니다.

## 핵심 원리

1. **피벗 선택**: 보통 마지막 원소. 최악 방지를 위해 랜덤/중간값/세 값의 중간값 선택 가능
2. **분할 (Partition)**: 피벗보다 작은 원소는 왼쪽, 큰 원소는 오른쪽으로 재배치
3. **재귀**: 피벗의 왼쪽과 오른쪽 부분을 각각 퀵 정렬로 정렬

## Partition 과정

```
피벗 = arr[high] (마지막 원소)
i = 작은 원소들의 경계 인덱스

low부터 high-1까지 순회:
  - arr[j] <= 피벗이면: i 증가, arr[i]와 arr[j] 교환

마지막에 arr[i+1]과 피벗 교환
→ 피벗의 최종 위치: i+1
```

### Partition 예시

`arr = [5, 2, 8, 1, 3]`, 피벗 = 3 (마지막 원소)

| j | arr[j] | 조건 | i | swap 후 arr |
|---|--------|------|---|-------------|
| 0 | 5 | 5>3 건너뜀 | -1 | [5,2,8,1,3] |
| 1 | 2 | 2≤3 | 0 | [2,5,8,1,3] (arr[0]↔arr[1]) |
| 2 | 8 | 8>3 건너뜀 | 1 | [2,5,8,1,3] |
| 3 | 1 | 1≤3 | 2 | [2,1,8,5,3] (arr[2]↔arr[3]) |

마지막: arr[i+1]과 피벗 교환 → arr[3]↔arr[4] → [2,1,3,5,8]

→ 피벗 3의 왼쪽 [2,1], 오른쪽 [5,8] 각각 재귀 정렬

## 시간 복잡도

| 구분 | 복잡도 |
|------|--------|
| 평균 | O(n log n) |
| 최선 | O(n log n) |
| 최악 | O(n²) - 이미 정렬된 배열에서 피벗이 최소/최대일 때 |

## 공간 복잡도

- O(log n) ~ O(n) (재귀 스택)

## 피벗 선택 전략

- **마지막 원소**: 구현 단순, 이미 정렬된 배열에서 O(n²)
- **랜덤**: 최악 확률 감소
- **중간값 of three**: 첫/중/끝 중 중간값 선택으로 실전에서 안정적

## 장단점

**장점**
- 평균적으로 매우 빠름
- 제자리 정렬(in-place) 가능

**단점**
- 최악의 경우 O(n²)
- 안정 정렬이 아님 (같은 값의 상대적 순서가 바뀔 수 있음)
