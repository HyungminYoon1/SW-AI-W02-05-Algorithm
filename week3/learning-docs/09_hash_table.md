# 해시 테이블 (Hash Table)

## 개요

**해시 테이블**은 Key-Value 쌍을 저장하고, Key를 통해 Value를 O(1)에 가까운 시간에 조회할 수 있는 자료구조입니다. 해시 함수를 사용해 Key를 인덱스로 변환합니다.

## 핵심 원리

1. **해시 함수**: Key → 해시값 (인덱스)
2. **버킷/슬롯**: 해시값에 해당하는 저장 공간
3. **충돌 해결**: 다른 Key가 같은 해시값을 가질 때 처리
   - **체이닝**: 같은 버킷에 연결 리스트로 여러 항목 저장
   - **개방 주소법**: 빈 슬롯을 찾을 때까지 선형/이차 탐사 등으로 다음 위치 탐색

## 파이썬에서의 구현

파이썬의 **딕셔너리(dict)**가 내부적으로 해시 테이블로 구현되어 있습니다. 따라서 딕셔너리를 사용하면 해시 테이블의 특성을 그대로 활용할 수 있습니다.

```python
students = {"Alice": 85, "Bob": 92, "Charlie": 78}
students["Alice"]      # 85 조회 - O(1)에 가까움
students["David"] = 90 # 삽입
del students["Bob"]    # 삭제
```

## 시간 복잡도 (평균)

| 연산 | 복잡도 |
|------|--------|
| 검색 | O(1) |
| 삽입 | O(1) |
| 삭제 | O(1) |

> 해시 함수 품질, 로드 팩터(저장률)에 따라 실제 성능이 달라짐. 충돌이 많으면 O(1)보다 느려질 수 있음

## 활용 예시

- 학생 성적 관리 (이름 → 점수)
- 단어 빈도수 계산
- 캐시 구현
- 중복 확인
- 두 수의 합 찾기 (보완 배열 활용)

## 성적 관리 예시

```python
# 평균: sum(scores.values()) / len(scores)
# 최고점 학생: max(scores, key=scores.get)
# 특정 학생 조회: scores.get(name)
```

## 장단점

**장점**
- 검색, 삽입, 삭제가 평균 O(1)
- Key-Value 매핑에 적합

**단점**
- 해시 충돌 가능
- 순서가 보장되지 않음 (Python 3.7+ dict는 삽입 순서 유지)
- 메모리 사용이 배열보다 많을 수 있음
