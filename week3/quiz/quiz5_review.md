5. 배열에서 특정 값을 찾기 위해 처음부터 끝까지 순차적으로 탐색하는 경우의 시간 복잡도를 빅오(Big-O) 표기법으로 작성하고, 그 이유를 함께 설명하시오.

---
# 작성한 응답
O(n) 
// 정렬이 되어있지 않고, 원하는 값이 어디에 있는지 모르므로, 앞에서부터 순차적으로 찾으려면 평균적으로 (n+1)/2 번의 탐색 횟수가 필요합니다.

---
# 올바른 답
O(n)

배열 크기를 n이라고 하면:
비교 횟수: 최대 n번
각 비교 연산: O(1)
따라서 전체 시간: O(1) + O(1) + ... + O(1) (n번) = O(n)

| 경우 | 시간 복잡도 | 설명                 |
| -- | ------ | ------------------ |
| 최선 | O(1)   | 첫 번째 원소에서 발견       |
| 평균 | O(n)   | 중간 위치에서 발견 (≈ n/2) |
| 최악 | O(n)   | 마지막 또는 없음          |
