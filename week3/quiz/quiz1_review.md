# 작성한 응답
서로 키 값이 다른 두 원소가 같은 해시함수 결과를 가질 수 있습니다. 이 때 충돌이 발생합니다.
학생과 사물함에 비유하자면, 학생은 고유하고 그 수가 많지만, 사물함의 수량은 제한되어 있습니다. 그래서 기수별, 학번별로 할당 후 매핑을 하여 사물함을 사용하도록 합니다. 그러나 종종 학생이 휴학/복학, 전과, 기타 모종의 사유로 할당된 기간/장소를 벗어나게 되면 서로 다른 두 학생이 동시에 같은 사물함을 사용하도록 매핑되는 결과가 발생할 수 있습니다.

--
# 올바른 답

## 1. 개념 정의
해시 충돌 (Hash Collision): 서로 다른 입력값이 같은 해시값(같은 저장 위치)을 가지는 현상

## 2. 비유 (학생–사물함)
학생 → 데이터 (key)
학번 → 해시 함수 입력값
사물함 번호 → 해시값 (저장 위치)

학교에서: “학번을 기준으로 사물함 번호를 자동 배정한다”

## 3. 충돌 발생 상황
서로 다른 학생이 같은 사물함(3번)을 배정받음
이 상황이 바로 해시 충돌

## 결론
학생을 학번으로 사물함에 배정할 때, 서로 다른 학생이 동일한 사물함 번호를 배정받는 경우가 발생하는데, 이와 같이 서로 다른 입력이 같은 저장 위치를 가지는 현상을 해시 충돌이라고 한다.
