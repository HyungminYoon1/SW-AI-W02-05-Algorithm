# 02. 이진 검색 트리

## BST란?

BST는 Binary Search Tree의 약자로, 한국어로는 이진 검색 트리 또는 이진 탐색 트리라고 합니다. 기본 형태는 이진 트리이지만, 여기에 "값의 크기 관계"라는 강한 규칙이 추가됩니다.

BST의 핵심 규칙은 다음과 같습니다.

- 왼쪽 서브트리의 모든 값은 현재 노드 값보다 작다.
- 오른쪽 서브트리의 모든 값은 현재 노드 값보다 크다.

이 규칙 덕분에 배열에서 이분 탐색을 하듯, 트리에서도 어느 방향으로 내려가야 할지 빠르게 결정할 수 있습니다.

## 예시로 이해하기

```text
      5
     / \
    3   7
   / \
  2   4
```

이 트리는 BST입니다.

- `5`의 왼쪽에는 `3`, `2`, `4`가 있고 모두 `5`보다 작습니다.
- `5`의 오른쪽에는 `7`이 있고 `5`보다 큽니다.
- `3`을 기준으로 보면 왼쪽 `2`는 작고 오른쪽 `4`는 큽니다.

즉, 트리 전체에서 규칙이 재귀적으로 유지됩니다.

## 왜 BST가 유용한가?

일반 이진 트리는 값의 순서에 대한 규칙이 없기 때문에 원하는 값을 찾으려면 전체를 다 뒤질 수도 있습니다. 하지만 BST는 왼쪽이 작은 값, 오른쪽이 큰 값이라는 규칙이 있으므로 탐색 범위를 절반씩 줄이는 느낌으로 내려갈 수 있습니다.

예를 들어 `4`를 찾는다고 해 보겠습니다.

1. 루트 `5`와 비교
2. `4 < 5`이므로 왼쪽으로 이동
3. 노드 `3`과 비교
4. `4 > 3`이므로 오른쪽으로 이동
5. 노드 `4`를 찾음

모든 노드를 볼 필요 없이, 비교 결과에 따라 한 방향만 선택하면 됩니다.

## 코드에서 보는 BST 검색

`week4/basic/02_bst.py`의 핵심 함수는 아래와 같습니다.

```python
def search_bst(root, target):
    if root is None:
        return False

    if target == root.value:
        return True
    elif target < root.value:
        return search_bst(root.left, target)
    elif target > root.value:
        return search_bst(root.right, target)
```

이 함수는 세 가지 경우만 판단합니다.

1. 현재 노드가 없으면 찾는 값도 없음
2. 현재 값과 같으면 찾음
3. 더 작으면 왼쪽, 더 크면 오른쪽으로 이동

정말 중요한 점은 "한 번 비교할 때마다 갈 방향이 결정된다"는 것입니다.

## 다른 사람에게 쉽게 설명하는 방법

BST를 설명할 때는 사전 찾기를 비유로 들면 좋습니다.

- 찾는 단어가 현재 페이지보다 앞에 있으면 앞쪽으로 이동
- 뒤에 있으면 뒤쪽으로 이동
- 정확히 같으면 찾음

BST도 마찬가지로 현재 노드와 비교해서 왼쪽인지 오른쪽인지 결정합니다.

## BST와 일반 이진 트리의 차이

- 이진 트리: 자식이 최대 2개라는 구조만 만족하면 됨
- BST: 자식이 최대 2개이면서, 왼쪽 < 부모 < 오른쪽 규칙까지 만족해야 함

즉, 모든 BST는 이진 트리이지만, 모든 이진 트리가 BST는 아닙니다.

## 검색 과정 단계별 예시

트리:

```text
      8
     / \
    3   10
   / \    \
  1   6    14
```

`6`을 찾는 과정:

1. `8`과 비교: `6 < 8`이므로 왼쪽
2. `3`과 비교: `6 > 3`이므로 오른쪽
3. `6`과 비교: 찾음

`13`을 찾는 과정:

1. `8`과 비교: 오른쪽
2. `10`과 비교: 오른쪽
3. `14`과 비교: 왼쪽이어야 하지만 노드가 없음
4. 따라서 값이 없음

## 시간 복잡도

평균적으로:

- 검색: `O(log n)`
- 삽입: `O(log n)`
- 삭제: `O(log n)`

하지만 트리가 한쪽으로 치우치면:

- 검색: `O(n)`

예를 들어 `1, 2, 3, 4, 5`를 차례대로 넣으면 트리가 거의 연결 리스트처럼 길게 늘어질 수 있습니다.

```text
1
 \
  2
   \
    3
     \
      4
```

이 경우 BST의 장점이 크게 줄어듭니다.

## 중위 순회와 BST의 관계

BST에서 중위 순회는 매우 중요합니다.

중위 순회 순서:

`왼쪽 -> 루트 -> 오른쪽`

BST는 왼쪽이 작고 오른쪽이 크므로, 중위 순회를 하면 값이 오름차순으로 나옵니다.

예를 들어 위 트리를 중위 순회하면:

```python
[2, 3, 4, 5, 7]
```

이 특징은 BST를 설명할 때 꼭 언급하면 좋습니다.

"BST는 중위 순회하면 정렬된 결과를 얻을 수 있다."

## 언제 쓰일까?

- 빠른 검색이 필요한 경우
- 정렬된 데이터 구조를 유지하고 싶은 경우
- 범위 검색이 필요한 경우
- 트리 기반 검색 구조를 배울 때 기초 개념으로

실무에서는 균형을 자동으로 맞추는 AVL 트리, Red-Black Tree 같은 구조가 더 자주 쓰이지만, 그 기반 개념은 BST입니다.

## 자주 하는 실수

- 부모와 직접 연결된 자식만 비교하고 끝내는 실수
- 왼쪽 전체 서브트리, 오른쪽 전체 서브트리의 규칙을 놓치는 실수
- 중복 값 처리 규칙을 정하지 않는 실수

현재 실습 코드는 검색만 다루므로 중복 값 규칙은 중요하지 않지만, 실제 BST 구현에서는 중복을 허용할지, 허용하면 어느 쪽에 넣을지 정해야 합니다.

## 다른 사람에게 가르칠 때의 핵심 포인트

- "BST는 값의 대소 관계를 이용해 검색 속도를 높인 트리입니다."
- "현재 값보다 작으면 왼쪽, 크면 오른쪽으로 이동합니다."
- "배열의 이분 탐색을 트리에서 하는 느낌이라고 생각하면 됩니다."
- "중위 순회를 하면 정렬된 순서가 나옵니다."

## 정리

BST는 이진 트리에 정렬 규칙을 더한 자료구조입니다. 이 규칙 덕분에 빠른 탐색이 가능해지고, 중위 순회를 통해 정렬된 결과도 얻을 수 있습니다. 트리와 탐색의 연결 고리를 이해하는 데 매우 중요한 개념이므로, 단순히 코드만 외우기보다 "왜 왼쪽 또는 오른쪽만 보면 되는지"를 꼭 이해하는 것이 중요합니다.
