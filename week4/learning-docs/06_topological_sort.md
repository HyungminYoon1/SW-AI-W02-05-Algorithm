# 06. 위상 정렬

## 위상 정렬이란?

위상 정렬(Topological Sort)은 방향 그래프에서 정점들을 "선행 관계를 어기지 않도록" 순서대로 나열하는 방법입니다.

쉽게 말하면:

- 어떤 작업은 다른 작업보다 먼저 해야 한다
- 어떤 과목은 다른 과목을 듣고 나서 들어야 한다

이런 "먼저 해야 하는 것"이 있는 상황에서 전체 순서를 정하는 알고리즘입니다.

## 언제 사용할까?

위상 정렬은 다음과 같은 상황에 자주 등장합니다.

- 선수 과목 순서 정하기
- 프로젝트 작업 순서 정하기
- 빌드 순서 정하기
- 의존성이 있는 작업 실행 순서 정하기

예를 들어:

- 기초 과목을 먼저 듣고
- 그다음 중급 과목을 듣고
- 마지막에 고급 과목을 듣는 구조

이런 관계를 정리할 때 위상 정렬을 사용합니다.

## 중요한 전제 조건

위상 정렬은 아무 그래프에서나 할 수 있는 것이 아닙니다.

반드시 다음 조건을 만족해야 합니다.

- 방향 그래프여야 한다.
- 사이클이 없어야 한다.

즉, DAG(Directed Acyclic Graph, 방향 비순환 그래프)에서만 가능합니다.

왜 사이클이 있으면 안 될까요?

예를 들어:

```text
A -> B -> C -> A
```

이 경우는:

- A를 하려면 C가 먼저여야 하고
- C를 하려면 B가 먼저여야 하고
- B를 하려면 A가 먼저여야 합니다.

서로 먼저 하라고 얽혀 있으므로 올바른 시작점이 없습니다.

## 핵심 개념: 진입 차수

위상 정렬에서 가장 중요한 개념은 진입 차수(in-degree)입니다.

진입 차수란:

"어떤 정점으로 들어오는 간선의 개수"

입니다.

예를 들어:

```text
0 -> 1
0 -> 2
1 -> 3
```

각 정점의 진입 차수는:

- `0`: 0
- `1`: 1
- `2`: 1
- `3`: 1

진입 차수가 0이라는 뜻은, 그 정점을 먼저 해도 되는 상태라는 뜻입니다. 즉, 아직 기다려야 할 선행 작업이 없다는 뜻입니다.

## Kahn 알고리즘이란?

실습 파일 `week4/basic/06_topological_sort.py`는 Kahn 알고리즘 방식의 위상 정렬을 의도하고 있습니다.

이 알고리즘의 흐름은 아래와 같습니다.

1. 그래프를 만든다.
2. 각 정점의 진입 차수를 계산한다.
3. 진입 차수가 0인 정점을 큐에 넣는다.
4. 큐에서 하나를 꺼내 결과에 넣는다.
5. 그 정점에서 나가는 간선을 제거한 것처럼 처리한다.
6. 연결된 다음 정점들의 진입 차수를 1씩 줄인다.
7. 새롭게 진입 차수가 0이 된 정점을 큐에 넣는다.
8. 큐가 빌 때까지 반복한다.

핵심 아이디어는 아주 직관적입니다.

"지금 바로 할 수 있는 일부터 처리하고, 그 일이 끝나면 다음에 할 수 있는 일을 연다."

## 예제로 이해하기

간선:

```python
[(0, 1), (0, 2), (1, 3)]
```

그래프로 그리면:

```text
0 -> 1 -> 3
 \ 
  -> 2
```

### 1. 진입 차수 계산

- `0`: 0
- `1`: 1
- `2`: 1
- `3`: 1

### 2. 진입 차수 0인 정점을 큐에 넣기

- 큐: `[0]`

### 3. `0` 처리

- 결과: `[0]`
- `0`에서 나가는 간선은 `0 -> 1`, `0 -> 2`
- 따라서 `1`, `2`의 진입 차수를 1씩 줄임

새 상태:

- `1`: 0
- `2`: 0
- `3`: 1

- 큐: `[1, 2]`

### 4. `1` 처리

- 결과: `[0, 1]`
- `1 -> 3`이 있으므로 `3`의 진입 차수 감소

새 상태:

- `3`: 0
- 큐: `[2, 3]`

### 5. `2` 처리

- 결과: `[0, 1, 2]`
- `2`에서 나가는 간선이 없으면 그대로 진행

### 6. `3` 처리

- 결과: `[0, 1, 2, 3]`

최종 위상 정렬 결과:

```python
[0, 1, 2, 3]
```

하지만 `[0, 2, 1, 3]`도 올바른 답입니다. 이유는 `1`과 `2` 사이에는 직접적인 선후 관계가 없기 때문입니다.

## 위상 정렬 결과는 하나가 아닐 수 있다

이 부분은 매우 중요합니다.

위상 정렬은 "정답이 하나만 존재하는 정렬"이 아닙니다. 선행 관계만 지키면 여러 순서가 가능합니다.

예를 들어:

- `0`이 먼저여야 한다
- `3`은 `1` 뒤여야 한다

이 규칙만 만족하면 `1`과 `2`의 상대 순서는 바뀔 수 있습니다.

## 코드로 옮기면 어떤 구조가 필요한가?

위상 정렬 구현에는 보통 아래 자료구조가 필요합니다.

- 그래프 인접 리스트
- 진입 차수 배열 또는 딕셔너리
- 진입 차수 0인 정점을 담는 큐
- 결과 순서를 저장할 리스트

구현 형태는 보통 아래와 비슷합니다.

```python
from collections import deque

def topological_sort(vertices, edges):
    graph = {i: [] for i in range(vertices)}
    in_degree = {i: 0 for i in range(vertices)}

    for u, v in edges:
        graph[u].append(v)
        in_degree[v] += 1

    q = deque()
    for node in range(vertices):
        if in_degree[node] == 0:
            q.append(node)

    result = []

    while q:
        current = q.popleft()
        result.append(current)

        for nxt in graph[current]:
            in_degree[nxt] -= 1
            if in_degree[nxt] == 0:
                q.append(nxt)

    return result
```

## 시간 복잡도

위상 정렬의 시간 복잡도는:

`O(V + E)`

- `V`: 정점 수
- `E`: 간선 수

각 정점은 큐에 들어가고 나오는 과정이 한 번씩 일어나고, 각 간선도 진입 차수를 줄일 때 한 번씩 확인되기 때문입니다.

## 사이클 판별과 연결되는 이유

위상 정렬을 수행했는데 결과 리스트의 길이가 전체 정점 수보다 작다면, 보통 사이클이 있다는 뜻으로 해석할 수 있습니다.

왜냐하면 사이클이 있으면 진입 차수가 0이 되는 정점이 더 이상 생기지 않기 때문입니다.

즉, 위상 정렬은 단순히 순서를 정하는 도구일 뿐 아니라, 방향 그래프의 사이클 여부를 확인하는 데도 활용될 수 있습니다.

## 자주 헷갈리는 포인트

- 위상 정렬은 방향 그래프에서만 의미가 있습니다.
- 무방향 그래프에는 선후 관계 개념이 없어서 위상 정렬을 적용하지 않습니다.
- 사이클이 있으면 위상 정렬이 성립하지 않습니다.
- 결과는 여러 개일 수 있습니다.
- 진입 차수 0인 정점부터 시작하는 이유는 지금 당장 수행 가능한 작업이기 때문입니다.

## 다른 사람에게 설명할 때 좋은 표현

- "위상 정렬은 먼저 해야 할 일을 먼저 오도록 줄 세우는 알고리즘입니다."
- "선수 과목이나 작업 의존성을 처리할 때 자주 쓰입니다."
- "진입 차수가 0이라는 것은 기다릴 선행 작업이 없다는 뜻입니다."
- "할 수 있는 작업부터 처리하면서 다음 작업을 열어 가는 방식입니다."

## 정리

위상 정렬은 방향 그래프의 선행 관계를 만족하는 순서를 만드는 알고리즘입니다. 핵심은 진입 차수와 큐이며, DAG에서만 사용할 수 있습니다. 이 개념을 제대로 이해하면 과목 순서, 작업 스케줄, 빌드 의존성 같은 문제를 자연스럽게 해석할 수 있고, 그래프 알고리즘의 활용 범위도 훨씬 넓어집니다.
