# 05. DFS

## DFS란?

DFS는 Depth-First Search의 약자로, 한국어로는 깊이 우선 탐색이라고 합니다. 이름 그대로 한 방향으로 가능한 만큼 깊게 들어갔다가, 더 이상 갈 곳이 없으면 되돌아와 다른 길을 탐색합니다.

쉽게 비유하면 미로에서 한 길을 정해서 끝까지 가 보고, 막히면 다시 돌아와 다른 길을 선택하는 방식과 비슷합니다.

## DFS의 핵심 생각

DFS는 현재 위치에서 갈 수 있는 곳이 있으면 먼저 그쪽으로 계속 이동합니다. 그래서 탐색 순서는 "깊이"를 우선합니다.

그래프:

```text
  0 --- 1
  |     |
  2 ----
  |
  3
```

시작 정점이 `0`이면, 구현에 따라 방문 순서는 달라질 수 있지만 실습 코드에서는 보통:

```python
[0, 1, 2, 3]
```

와 같은 순서가 나옵니다.

중요한 점은 BFS와 달리 DFS는 "가까운 순서"를 보장하지 않는다는 것입니다.

## 재귀로 구현하는 이유

DFS는 재귀와 아주 잘 어울립니다. 현재 정점을 방문한 뒤, 방문하지 않은 인접 정점이 있으면 그 정점으로 들어가 같은 작업을 반복하면 되기 때문입니다.

`week4/basic/05_dfs.py`의 핵심 함수는 아래와 같습니다.

```python
def dfs(graph, start, visited=None):
    if visited is None:
        visited = []

    visited.append(start)

    for next_node in graph.get(start, []):
        if next_node not in visited:
            dfs(graph, next_node, visited=visited)

    return visited
```

이 코드의 흐름은 매우 단순합니다.

1. 현재 정점을 방문한다.
2. 인접 정점을 하나씩 본다.
3. 아직 방문하지 않았다면 그 정점으로 재귀 호출한다.
4. 더 이상 갈 곳이 없으면 자동으로 이전 호출로 돌아간다.

## 단계별로 따라가 보기

그래프:

```python
{
    0: [1, 2],
    1: [0, 2],
    2: [0, 1, 3],
    3: [2]
}
```

`dfs(graph, 0)` 실행:

### 1단계

- `0` 방문
- 방문 목록: `[0]`
- 이웃 `1`, `2` 중 먼저 `1`로 이동

### 2단계

- `1` 방문
- 방문 목록: `[0, 1]`
- 이웃 `0`, `2` 확인
- `0`은 이미 방문했으므로 넘어감
- `2`로 이동

### 3단계

- `2` 방문
- 방문 목록: `[0, 1, 2]`
- 이웃 `0`, `1`, `3` 확인
- `0`, `1`은 이미 방문
- `3`으로 이동

### 4단계

- `3` 방문
- 방문 목록: `[0, 1, 2, 3]`
- 더 이상 새로 갈 곳이 없음

이제 재귀 호출이 끝나면서 차례대로 이전 위치로 돌아갑니다.

## DFS의 특징

### 1. 한 경로를 끝까지 본다

DFS는 현재 갈 수 있는 길을 먼저 끝까지 확인합니다. 그래서 경로 탐색이나 경우의 수 탐색과 잘 맞습니다.

### 2. 재귀 또는 스택으로 구현할 수 있다

재귀는 내부적으로 호출 스택을 사용합니다. 따라서 DFS는 재귀 구현과 스택 구현이 본질적으로 비슷합니다.

### 3. 방문 체크가 필요하다

그래프에는 사이클이 있을 수 있기 때문에, 방문 여부를 기록하지 않으면 같은 노드를 계속 방문할 수 있습니다.

## BFS와 DFS의 차이

- BFS: 가까운 곳부터 넓게 탐색
- DFS: 한 방향으로 깊게 탐색

비유로 설명하면:

- BFS는 파도가 퍼지듯 이동
- DFS는 동굴 한 갈래를 끝까지 따라감

## DFS가 자주 쓰이는 상황

- 경로가 존재하는지 확인
- 모든 경우의 수 탐색
- 연결 요소 찾기
- 사이클 탐지
- 백트래킹 문제의 기초

특히 재귀와 함께 쓰이기 때문에, DFS를 잘 이해하면 백트래킹과 트리 탐색도 훨씬 쉬워집니다.

## 시간 복잡도

인접 리스트 기준으로:

`O(V + E)`

- `V`: 정점 수
- `E`: 간선 수

각 정점과 간선을 필요한 만큼 확인하기 때문입니다.

## 현재 실습 코드의 특징과 주의점

실습 코드는 `visited`를 리스트로 관리합니다. 교육용으로는 흐름을 이해하기 좋지만, 방문 여부 확인을 자주 해야 하는 큰 그래프에서는 집합(`set`)을 함께 쓰는 방식이 더 효율적일 수 있습니다.

예를 들어:

- 리스트에서 `x in visited`는 느릴 수 있음
- 집합에서 `x in visited_set`은 더 빠름

하지만 입문 단계에서는 리스트 기반 구현이 동작 원리를 이해하는 데 충분히 좋습니다.

## 자주 하는 실수

- 방문 체크 없이 재귀 호출해서 무한 반복되는 경우
- `visited=None` 초기화를 하지 않아 함수 기본값 문제를 만드는 경우
- DFS 결과 순서가 항상 하나라고 생각하는 경우

그래프의 인접 리스트 순서에 따라 DFS 방문 순서는 달라질 수 있습니다. 따라서 "DFS의 원리"가 중요하지, 항상 똑같은 결과 순서가 중요한 것은 아닙니다.

## 다른 사람에게 설명할 때 좋은 문장

- "DFS는 한 방향으로 갈 수 있을 때까지 끝까지 들어가는 탐색입니다."
- "막히면 되돌아와 다른 길을 찾습니다."
- "재귀 함수와 매우 잘 어울리는 그래프 탐색 방식입니다."
- "방문 체크가 없으면 같은 노드를 계속 돌 수 있습니다."

## BFS와 함께 외우면 좋은 비교

- BFS는 큐
- DFS는 재귀 또는 스택
- BFS는 최단 거리 문제에 강함
- DFS는 경로 탐색과 구조 분석에 강함

## 정리

DFS는 그래프와 트리를 깊게 탐색하는 대표 알고리즘입니다. 한 방향으로 끝까지 들어갔다가 돌아오는 흐름을 이해하면 재귀의 동작 원리도 함께 익힐 수 있습니다. BFS와 비교하면서 배우면 차이가 더 선명해지고, 이후 백트래킹과 고급 탐색 문제를 공부할 때도 큰 도움이 됩니다.
