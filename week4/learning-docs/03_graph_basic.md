# 03. 그래프 기본

## 그래프란?

그래프(Graph)는 여러 개의 대상이 서로 어떻게 연결되어 있는지를 표현하는 자료구조입니다. 트리가 계층 구조를 나타내는 데 강하다면, 그래프는 훨씬 더 자유로운 연결 관계를 표현하는 데 강합니다.

그래프는 보통 두 가지 요소로 이루어집니다.

- 정점(Vertex): 하나의 대상
- 간선(Edge): 정점과 정점을 잇는 연결

예를 들어:

- 사람과 친구 관계
- 도시와 도로 연결
- 컴퓨터와 네트워크 연결
- 과목과 선수 과목 관계

이런 것들은 모두 그래프로 표현할 수 있습니다.

## 그래프와 트리의 차이

트리는 그래프의 특별한 형태입니다. 그래프는 훨씬 자유롭기 때문에 다음과 같은 특징이 있습니다.

- 사이클이 있을 수 있음
- 부모-자식 개념이 꼭 필요하지 않음
- 한 노드가 여러 노드와 자유롭게 연결될 수 있음
- 연결되지 않은 정점이 있을 수도 있음

즉, 그래프는 현실의 복잡한 관계를 다루기 위한 일반적인 모델입니다.

## 방향 그래프와 무방향 그래프

그래프는 간선에 방향이 있는지에 따라 두 가지로 나뉩니다.

### 무방향 그래프

정점 A와 B가 연결되어 있으면 양쪽으로 갈 수 있습니다.

예:

`0 -- 1`

이는 `0`에서 `1`로도 갈 수 있고, `1`에서 `0`으로도 갈 수 있다는 뜻입니다.

친구 관계, 양방향 도로 같은 경우에 어울립니다.

### 방향 그래프

간선이 한쪽 방향만 가리킵니다.

예:

`0 -> 1`

이는 `0`에서 `1`로는 갈 수 있지만, 반대로는 자동으로 갈 수 없다는 뜻입니다.

팔로우 관계, 선수 과목, 작업 순서 같은 경우에 어울립니다.

## 그래프를 표현하는 방법

그래프는 여러 방식으로 저장할 수 있지만, 입문 단계에서는 인접 리스트(Adjacency List)를 가장 많이 씁니다.

`week4/basic/03_graph_basic.py`도 인접 리스트를 사용합니다.

## 인접 리스트란?

각 정점마다 "내가 연결된 이웃 정점들"을 리스트로 저장하는 방식입니다.

예를 들어 간선이 아래와 같다고 합시다.

```python
[(0, 1), (0, 2), (1, 2), (2, 3)]
```

무방향 그래프라면:

```python
{
    0: [1, 2],
    1: [0, 2],
    2: [0, 1, 3],
    3: [2]
}
```

이 구조는 "정점 0은 1과 2에 연결되어 있다"처럼 읽을 수 있어 매우 직관적입니다.

## 코드 이해하기

실습 코드의 핵심 함수는 아래와 같습니다.

```python
def create_graph(vertices, edges, directed=False):
    graph = {}
    for i in range(vertices):
        graph[i] = []

    for u, v in edges:
        graph[u].append(v)
        if not directed:
            graph[v].append(u)

    return graph
```

이 함수는 다음 순서로 동작합니다.

1. 정점 수만큼 빈 리스트를 준비합니다.
2. 간선을 하나씩 읽습니다.
3. `u -> v`를 추가합니다.
4. 무방향 그래프면 `v -> u`도 같이 추가합니다.

## 왜 딕셔너리와 리스트를 같이 쓸까?

- 딕셔너리: 각 정점 번호를 빠르게 찾기 좋음
- 리스트: 그 정점과 연결된 이웃들을 순서대로 저장하기 좋음

그래서 `정점 -> 연결된 정점들` 구조를 만들기 쉬워집니다.

## 예시를 단계별로 따라가 보기

정점 4개, 간선:

```python
[(0, 1), (0, 2), (1, 2), (2, 3)]
```

초기 상태:

```python
{
    0: [],
    1: [],
    2: [],
    3: []
}
```

간선 `(0, 1)` 추가 후:

```python
{
    0: [1],
    1: [0],
    2: [],
    3: []
}
```

간선 `(0, 2)` 추가 후:

```python
{
    0: [1, 2],
    1: [0],
    2: [0],
    3: []
}
```

이렇게 계속 채워 나가면 최종 그래프가 완성됩니다.

## 그래프를 배울 때 중요한 이유

그래프는 알고리즘에서 매우 자주 등장합니다. BFS, DFS, 최단 경로, 최소 신장 트리, 위상 정렬 등 많은 핵심 알고리즘이 그래프 위에서 동작합니다.

그래서 그래프 입문에서는 먼저 이 질문에 익숙해지는 것이 중요합니다.

- 정점이 무엇인가?
- 간선이 무엇인가?
- 방향이 있는가?
- 어떻게 저장할 것인가?

## 시간 복잡도 관점

인접 리스트의 장점:

- 연결된 이웃만 저장하므로 메모리를 아끼기 좋음
- BFS, DFS처럼 인접 노드 탐색이 쉬움

그래프의 모든 정점과 간선을 기준으로 많이 표현하는 복잡도는:

- 정점 수: `V`
- 간선 수: `E`

인접 리스트는 보통 `O(V + E)` 크기의 저장 공간이 필요합니다.

## 자주 헷갈리는 포인트

- 무방향 그래프는 간선을 양쪽에 모두 넣어야 합니다.
- 방향 그래프는 한쪽만 넣습니다.
- 정점 번호가 꼭 0부터 시작할 필요는 없지만, 실습에서는 보통 0부터 시작합니다.
- 그래프의 모양을 머릿속 그림으로만 이해하지 말고, 딕셔너리 구조로도 읽을 수 있어야 합니다.

## 다른 사람에게 설명할 때 좋은 표현

- "그래프는 대상들과 그 사이의 연결 관계를 표현하는 자료구조입니다."
- "트리가 가족관계처럼 위아래가 분명한 구조라면, 그래프는 친구관계처럼 자유로운 연결 구조입니다."
- "인접 리스트는 각 정점에 대해 연결된 이웃들을 적어 두는 방식입니다."
- "무방향 그래프는 양쪽에 모두 기록하고, 방향 그래프는 한쪽만 기록합니다."

## 정리

그래프는 복잡한 연결 관계를 표현하는 가장 기본적인 자료구조입니다. 정점과 간선, 방향성, 인접 리스트를 이해하면 이후 BFS, DFS, 위상 정렬 같은 그래프 알고리즘을 훨씬 수월하게 배울 수 있습니다. 그래프를 처음 배울 때는 그림과 코드 표현을 함께 연결해서 이해하는 것이 가장 중요합니다.
