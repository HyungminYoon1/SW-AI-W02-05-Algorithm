# 01. 이진 트리

## 이진 트리란?

이진 트리(Binary Tree)는 각 노드가 최대 2개의 자식 노드를 가질 수 있는 트리 구조입니다. 자식은 보통 `왼쪽 자식(left)`과 `오른쪽 자식(right)`으로 구분합니다.

트리는 계층 구조를 표현하기에 좋습니다. 예를 들어 폴더 구조, 조직도, 게임의 선택지, 수식 구조 등을 표현할 때 자주 등장합니다. 이진 트리는 그중에서도 가장 기본이 되는 트리 형태라서 이후 배우는 BST, 힙, 세그먼트 트리 같은 구조의 출발점이 됩니다.

## 트리에서 먼저 알아야 할 말들

- 노드(Node): 데이터를 저장하는 하나의 단위
- 루트(Root): 트리의 맨 위 시작 노드
- 자식(Child): 어떤 노드 아래에 연결된 노드
- 부모(Parent): 자식을 가지는 위쪽 노드
- 리프(Leaf): 자식이 없는 노드
- 서브트리(Subtree): 특정 노드를 루트로 하는 작은 트리
- 깊이(Depth): 루트에서 해당 노드까지 내려간 간선 수

아래 구조를 예로 생각해 보면 이해가 쉽습니다.

```text
      1
     / \
    2   3
   / \
  4   5
```

- 루트는 `1`
- `1`의 왼쪽 자식은 `2`, 오른쪽 자식은 `3`
- `4`, `5`, `3`은 리프 노드입니다.

## 왜 중요한가?

이진 트리는 단순히 노드를 연결해 두는 구조가 아닙니다. "데이터를 어떤 순서로 방문할 것인가"를 배우는 첫 단계입니다. 트리 순회를 이해하면 이후 재귀, 탐색, 계층 구조 처리, 트리 기반 알고리즘을 훨씬 쉽게 배울 수 있습니다.

## 코드 구조 이해하기

`week4/basic/01_binary_tree.py`에서는 `TreeNode` 클래스로 노드를 만들고, 세 가지 순회 함수를 구현합니다.

```python
class TreeNode:
    def __init__(self, value):
        self.value = value
        self.left = None
        self.right = None
```

이 코드는 노드 하나가 값 하나와 왼쪽, 오른쪽 자식을 가질 수 있도록 만든 것입니다.

- `value`: 현재 노드의 값
- `left`: 왼쪽 자식을 가리킴
- `right`: 오른쪽 자식을 가리킴

이렇게 노드들이 연결되면 하나의 트리가 됩니다.

## 트리 순회란?

순회(Traversal)는 트리에 있는 모든 노드를 일정한 규칙에 따라 한 번씩 방문하는 것입니다.

이진 트리에서 가장 기본이 되는 순회는 세 가지입니다.

1. 전위 순회(Preorder)
2. 중위 순회(Inorder)
3. 후위 순회(Postorder)

핵심은 "루트를 언제 방문하느냐"입니다.

## 1. 전위 순회

방문 순서:

`루트 -> 왼쪽 서브트리 -> 오른쪽 서브트리`

예시 트리에 적용하면:

```text
1 -> 2 -> 4 -> 5 -> 3
```

결과:

```python
[1, 2, 4, 5, 3]
```

전위 순회는 "현재 노드를 먼저 처리하고 아래로 내려간다"는 느낌입니다. 트리 구조를 복사하거나, 계층 구조를 위에서부터 읽어 내려갈 때 자주 사용됩니다.

코드 흐름은 아래와 같습니다.

```python
def preorder(root):
    if root is None:
        return []

    result = []
    result.append(root.value)
    result.extend(preorder(root.left))
    result.extend(preorder(root.right))
    return result
```

설명할 때는 이렇게 말하면 좋습니다.

"전위 순회는 부모를 먼저 보고, 그다음 왼쪽, 마지막으로 오른쪽을 본다."

## 2. 중위 순회

방문 순서:

`왼쪽 서브트리 -> 루트 -> 오른쪽 서브트리`

예시 트리에 적용하면:

```text
4 -> 2 -> 5 -> 1 -> 3
```

결과:

```python
[4, 2, 5, 1, 3]
```

중위 순회는 특히 BST에서 매우 중요합니다. BST에서는 중위 순회를 하면 값이 오름차순으로 나옵니다. 그래서 단순한 순회 방식처럼 보여도, 정렬과 연결되는 강력한 의미를 갖습니다.

## 3. 후위 순회

방문 순서:

`왼쪽 서브트리 -> 오른쪽 서브트리 -> 루트`

예시 트리에 적용하면:

```text
4 -> 5 -> 2 -> 3 -> 1
```

결과:

```python
[4, 5, 2, 3, 1]
```

후위 순회는 자식들을 먼저 처리한 다음 부모를 처리합니다. 그래서 폴더 삭제, 수식 계산, 하위 작업 완료 후 상위 작업 처리 같은 상황에 잘 어울립니다.

## 재귀로 구현하는 이유

트리는 구조 자체가 재귀적입니다. 어떤 노드를 기준으로 보면 왼쪽도 트리이고, 오른쪽도 트리입니다. 그래서 "현재 노드를 처리하고, 왼쪽 트리를 처리하고, 오른쪽 트리를 처리한다"라는 방식이 자연스럽게 재귀와 맞아떨어집니다.

이 점은 다른 사람에게 설명할 때 아주 중요합니다.

"트리 문제에서 재귀가 자주 나오는 이유는 트리의 각 부분이 또 하나의 작은 트리이기 때문이다."

## 순회를 단계별로 이해해 보기

전위 순회로 예시 트리를 따라가 보면:

1. `1` 방문
2. 왼쪽으로 가서 `2` 방문
3. 왼쪽으로 가서 `4` 방문
4. `4`는 자식이 없으므로 돌아감
5. `2`의 오른쪽 `5` 방문
6. 다시 루트 `1`로 돌아와 오른쪽 `3` 방문

이 과정을 보면 재귀가 사실상 "내려가고, 막히면 돌아오는 과정"임을 알 수 있습니다.

## 시간 복잡도와 공간 복잡도

- 시간 복잡도: `O(n)`
- 공간 복잡도: `O(h)`

여기서:

- `n`: 전체 노드 수
- `h`: 트리의 높이

모든 노드를 한 번씩 방문하므로 시간은 `O(n)`입니다. 재귀 호출 스택은 트리 높이만큼 쌓일 수 있으므로 공간은 `O(h)`입니다.

## 자주 헷갈리는 포인트

- 이진 트리는 "최대 2개 자식"이지 "항상 2개 자식"이 아닙니다.
- 왼쪽 자식과 오른쪽 자식은 순서가 있는 개념입니다.
- 순회는 노드를 "어떤 규칙으로 방문하느냐"에 대한 개념입니다.
- 중위 순회가 항상 정렬 결과를 주는 것은 아닙니다. BST일 때만 그렇습니다.

## 실생활 비유

- 전위 순회: 회사 조직도에서 팀장을 먼저 소개하고 팀원들을 소개하는 방식
- 중위 순회: 왼쪽 자료를 보고 현재 자료를 보고 오른쪽 자료를 보는 방식
- 후위 순회: 하위 업무를 끝낸 뒤 상위 업무를 마무리하는 방식

## 다른 사람에게 설명할 때의 핵심 문장

- "이진 트리는 각 노드가 최대 두 개의 자식을 가지는 트리입니다."
- "트리 순회는 모든 노드를 한 번씩 방문하는 방법입니다."
- "전위, 중위, 후위의 차이는 루트를 언제 방문하느냐입니다."
- "트리는 부분 구조도 트리이기 때문에 재귀로 구현하기 좋습니다."

## 정리

이진 트리는 트리 자료구조의 가장 기본이 되는 형태입니다. 순회를 이해하면 트리 문제의 절반은 이미 익힌 셈입니다. 특히 전위, 중위, 후위 순회를 구분해서 이해하는 것이 중요하며, 이 개념은 이후 BST와 다양한 고급 트리 문제의 기반이 됩니다.
