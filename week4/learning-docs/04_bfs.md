# 04. BFS

## BFS란?

BFS는 Breadth-First Search의 약자로, 한국어로는 너비 우선 탐색이라고 합니다. 이름 그대로 한 지점에서 시작해서 "가까운 곳부터 넓게" 탐색하는 방식입니다.

쉽게 말하면, 한 정점에서 출발했을 때:

- 먼저 바로 연결된 정점들
- 그다음 한 단계 더 떨어진 정점들
- 그다음 두 단계 떨어진 정점들

이런 식으로 거리 순서대로 퍼져 나갑니다.

## 왜 큐를 사용할까?

BFS의 핵심은 먼저 들어온 정점을 먼저 처리하는 것입니다. 그래서 FIFO 구조인 큐(Queue)가 잘 맞습니다.

비유하면 줄 서기와 비슷합니다.

- 먼저 줄 선 사람이 먼저 나감
- 나중에 온 사람은 뒤에서 기다림

BFS도 시작점에서 가까운 정점들이 먼저 큐에 들어가고, 그 순서대로 처리됩니다.

## 예시 그래프

```text
  0 --- 1
  |     |
  2 ----
  |
  3
```

인접 리스트로 표현하면:

```python
{
    0: [1, 2],
    1: [0, 2],
    2: [0, 1, 3],
    3: [2]
}
```

시작 정점이 `0`이면 BFS 방문 순서는:

```python
[0, 1, 2, 3]
```

## 코드 이해하기

`week4/basic/04_bfs.py`에서는 `collections.deque`를 사용합니다.

```python
from collections import deque

def bfs(graph, start):
    visited = []
    visited_set = set()

    q = deque()
    q.append(start)
    visited_set.add(start)

    while q:
        node = q.popleft()
        visited.append(node)

        for picked_node in graph.get(node, []):
            if picked_node not in visited_set:
                q.append(picked_node)
                visited_set.add(picked_node)

    return visited
```

이 코드는 "큐에 넣을 때 방문 표시"를 하는 방식입니다. 이 방식은 같은 노드가 중복으로 큐에 들어가는 것을 막아 줍니다.

## 동작 과정을 단계별로 보기

시작 상태:

- 큐: `[0]`
- 방문 집합: `{0}`
- 방문 순서: `[]`

### 1단계

- 큐에서 `0`을 꺼냄
- 방문 순서에 `0` 추가
- 이웃 `1`, `2`를 확인
- 둘 다 아직 방문하지 않았으므로 큐에 넣음

현재 상태:

- 큐: `[1, 2]`
- 방문 집합: `{0, 1, 2}`
- 방문 순서: `[0]`

### 2단계

- 큐에서 `1`을 꺼냄
- 방문 순서에 `1` 추가
- 이웃 `0`, `2`는 이미 방문했으므로 넘어감

현재 상태:

- 큐: `[2]`
- 방문 순서: `[0, 1]`

### 3단계

- 큐에서 `2`를 꺼냄
- 방문 순서에 `2` 추가
- 이웃 중 `3`이 아직 방문되지 않았으므로 큐에 추가

현재 상태:

- 큐: `[3]`
- 방문 순서: `[0, 1, 2]`

### 4단계

- 큐에서 `3`을 꺼냄
- 방문 순서에 `3` 추가

최종 결과:

```python
[0, 1, 2, 3]
```

## BFS의 핵심 특징

### 1. 가까운 노드부터 탐색한다

시작점에서 간선 수 기준으로 가까운 노드부터 처리합니다. 그래서 최단 거리 문제의 기초가 됩니다.

### 2. 큐를 사용한다

먼저 발견한 노드를 먼저 처리해야 하므로 큐가 잘 맞습니다.

### 3. 방문 체크가 중요하다

그래프에는 사이클이 있을 수 있으므로, 방문 체크가 없으면 같은 노드를 계속 돌 수 있습니다.

## 시간 복잡도

인접 리스트를 기준으로 BFS의 시간 복잡도는:

`O(V + E)`

- `V`: 정점 수
- `E`: 간선 수

각 정점은 한 번씩 큐에서 나오고, 각 간선도 확인되기 때문입니다.

## 어디에 활용될까?

- 최단 거리 탐색
- 미로 탐색
- 네트워크에서 최소 단계 계산
- 친구 추천에서 가까운 관계 찾기
- 레벨 단위 탐색이 필요한 문제

예를 들어 가중치가 없는 그래프에서 시작점에서 다른 노드까지의 최소 이동 횟수를 구할 때 BFS가 자주 쓰입니다.

## DFS와 비교하면?

- BFS: 가까운 곳부터 넓게 탐색
- DFS: 한 방향으로 깊게 탐색

설명할 때는 이렇게 말하면 좋습니다.

"BFS는 물이 주변으로 퍼지듯 탐색하고, DFS는 한 길로 끝까지 들어갔다가 돌아오는 느낌입니다."

## 자주 하는 실수

- 방문 체크를 안 해서 무한 반복되는 경우
- 큐 대신 리스트를 쓰고 비효율적으로 앞에서 꺼내는 경우
- 방문 표시를 너무 늦게 해서 같은 노드가 여러 번 큐에 들어가는 경우

현재 실습 코드처럼 큐에 넣는 순간 방문 처리하면 중복 삽입을 줄일 수 있습니다.

## 다른 사람에게 가르칠 때 핵심 문장

- "BFS는 시작점에서 가까운 정점부터 차례대로 탐색하는 알고리즘입니다."
- "큐를 사용해서 먼저 발견한 정점을 먼저 처리합니다."
- "그래프에서 최단 거리 문제의 기본 도구입니다."
- "사이클이 있을 수 있으므로 방문 체크가 꼭 필요합니다."

## 정리

BFS는 그래프 탐색의 가장 기본적인 알고리즘 중 하나입니다. 가까운 정점부터 차례로 방문하므로 구조를 넓게 파악하기에 좋고, 특히 가중치가 없는 그래프에서 최단 거리를 구할 때 매우 강력합니다. 큐와 방문 체크의 역할을 정확히 이해하면 BFS는 훨씬 쉽게 다가옵니다.
